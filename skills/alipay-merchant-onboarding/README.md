# alipay-merchant-onboarding

## Overview

Alipay Open Platform Merchant Onboarding Skill — A one-stop solution for individual developers to become Alipay merchants, providing AI agents with full-process onboarding guidance including product recommendation, category selection, data collection, product signing, and application publishing.

## Features

- **Smart Product Recommendation** — Automatically recommends suitable payment products (PC Website Payment/AI Pay) based on user business scenarios
- **Intelligent MCC Matching** — Semantically matches business categories from user descriptions
- **OAuth Authorization** — Secure OAuth login via `alipay-cli` with user confirmation
- **Data Collection** — Collects screenshots for PC Website Payment or service registration info for AI Pay
- **Product Signing** — Submits signing applications through MCP services
- **Application Publishing** — Creates applications, sets public keys, and submits for review
- **Service Marketplace** — Registers AI Pay services to Alipay Service Marketplace
- **Multi-tool Detection** — Automatically detects AI coding tools (Claude Code, Cursor, Codex, etc.)

## Supported Payment Products

| Product | Code | Scenario | Data Required |
|---------|------|----------|---------------|
| PC Website Payment | `I1080300001000041203` | PC e-commerce, online education, government payment | 3 screenshots (homepage, product page, payment page) |
| AI Pay | `I1080300001000160457` | AI agents, API services, digital content, compute resources | None (service info collected interactively in Step 5.2) |

## Onboarding Flow

```
Step 1: Environment Check
    ↓
Step 2: Solution Planning (Product + MCC Recommendation)
    ↓
Step 3: Login Authorization (OAuth)
    ↓
Step 3.1: Post-Authorization (Status Query → Determine Signing Status)
    ├─ NOT_SIGNED → Step 4: Data Collection → Step 5
    └─ SIGNED → Skip data collection, go directly to Step 5
                      (AI Pay still needs service registration)
    ↓
Step 5: Onboarding Progress (Signing + Service Registration + App Publishing)
    ↓
Step 6: Flow Complete
```

## Security Guidelines

- **Private key must not be generated by the skill** — Users must generate keys using the official Alipay tool
- **Private key must not be stored in client** — Keep private keys secure
- **Private key must not appear in logs** — Never log sensitive key material
- **Private key must not be uploaded to public repositories** — Use `.gitignore`
- **Public key confirmation required** — Users must scan QR code to confirm public key settings
- **Authorization link format** — Always use the correct authorization URL format, never expose CLI-returned `verification_url`

## 概述

支付宝开放平台商家入驻技能 —— 个人开发者支付宝商家入驻一站式解决方案，为 AI 智能体提供产品推荐、类目选择、资料采集、产品签约、应用创建全流程指导。

## 功能特性

- **智能产品推荐** — 根据用户业务场景自动推荐合适的支付产品（电脑网站支付/AI收）
- **智能 MCC 匹配** — 根据用户描述语义匹配经营类目
- **OAuth 授权登录** — 通过 `alipay-cli` 安全登录，用户扫码确认
- **资料采集** — 电脑网站支付采集 3 张截图，AI收无需资料采集（服务信息在 Step 5.2 交互式收集）
- **产品签约** — 通过 MCP 服务提交签约申请
- **应用发布** — 创建应用、设置公钥、提交审核
- **服务市场上架** — AI收产品上架到支付宝服务市场
- **多工具识别** — 自动检测用户使用的 AI 编程工具（Claude Code、Cursor、Codex 等）

## 支持的支付产品

| 产品 | 产品码 | 适用场景 | 所需资料 |
|------|--------|----------|----------|
| 电脑网站支付 | `I1080300001000041203` | PC 电商、在线教育、政务缴费 | 3 张截图（首页、商品页、支付页） |
| AI收 | `I1080300001000160457` | AI 智能体、API 服务、数字内容、算力资源 | 无（服务信息在 Step 5.2 交互式收集） |

## 入驻流程

```
Step 1: 环境检查
    ↓
Step 2: 方案规划（产品推荐 + MCC 推荐）
    ↓
Step 3: 登录授权（OAuth）
    ↓
Step 3.1: 授权后处理（状态查询 → 判断签约状态）
    ├─ NOT_SIGNED → Step 4: 资料采集 → Step 5
    └─ SIGNED → 跳过资料采集，直接进入 Step 5
                      （AI收仍需服务注册）
    ↓
Step 5: 入驻推进（签约 + 服务注册 + 应用发布）
    ↓
Step 6: 流程结束
```

## 安全红线

- **禁止自动生成密钥** — 必须引导用户使用官方密钥生成工具
- **私钥禁止存储在客户端** — 妥善保管私钥
- **私钥禁止出现在日志中** — 禁止日志记录敏感密钥信息
- **私钥禁止上传到公开仓库** — 使用 `.gitignore` 排除
- **公钥设置需用户确认** — 用户必须扫码确认公钥设置
- **授权链接格式规范** — 使用正确的授权链接格式，禁止透出 CLI 返回的 `verification_url`
