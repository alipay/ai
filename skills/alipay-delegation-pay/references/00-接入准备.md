# 接入准备（陌生商户从零开始）

> 接入任何委托支付接口**之前**，必须先完成本文的准备工作。否则会卡在"没有 appId / 没有密钥 / 没有产品权限"无法调用任何接口。
> 来源：支付宝开放平台「智能体代理支付 - 接入准备」邀测文档 <https://opendocs.alipay.com/pre-open/0h6n92>

## 0. 接入方式选择

| 方式 | 适用 | 说明 |
|------|------|------|
| **商家自研** | 自己有研发 | 自助创建应用、开通产品、调接口（本文主线） |
| **服务商代开发** | 由 ISV 代接 | 服务商获取商家代开发授权后，用 `app_auth_token` 代调接口 |

---

## 1. 创建应用，获取 APPID

1. 登录 [支付宝开放平台](https://open.alipay.com)。
2. 创建「网页/移动应用」或「小程序应用」。
3. 创建后得到应用唯一标识 **APPID**（形如 `2021000xxxxxxxxx`），用于调用所有接口。

> 获取 APPID 指引：<https://opendocs.alipay.com/common/0i24qo>

---

## 2. 配置应用加签（生成密钥）—— 资金接口必做

委托支付涉及资金支出，**接口加签是必须的**。推荐先用「公钥模式」跑通沙箱，生产环境建议「证书模式」。

### 生成 RSA2 密钥对

1. 下载官方 [密钥生成工具](https://opendocs.alipay.com/common/02kipl)。
2. 生成 **RSA2（2048 位）** 密钥对，得到：**应用私钥** 和 **应用公钥**。
3. 在开放平台「开发设置 > 接口加签方式」**填入应用公钥**，保存后平台返回 **支付宝公钥**。

最终你会拿到 3 样东西，对应 SDK 初始化参数：

| 你拿到的 | 对应 SDK 参数 | 谁生成 |
|---------|--------------|--------|
| 应用私钥（PKCS8，妥善保管，勿泄露） | `privateKey` | 你自己 |
| 支付宝公钥 | `alipayPublicKey` | 支付宝 |
| APPID | `appId` | 支付宝（建应用时） |

> 接口加签方式：<https://opendocs.alipay.com/common/02mriz> ｜ 证书加签：<https://opendocs.alipay.com/common/056zub>

### 其他应用配置（开发 > 开发设置）

| 配置项 | 必填 | 说明 |
|--------|------|------|
| 接口加签方式 | 必填 | 资金接口必须设置；推荐生产用**证书加签** |
| 应用网关 | 必填 | 接收支付宝异步通知，需 **公网可访问的 http(s) 地址** |
| 服务器 IP 白名单 | 选填 | 提高安全性，避免私钥泄漏被滥用 |
| 授权回调地址 | 选填 | 网页/移动应用授权回调 URL |

## 3. 开通「智能体代理支付」产品 —— 邀测产品，必做

> ⚠️ **委托支付/AI付目前处于「定向开放（邀测）」阶段。应用上线后，必须联系支付宝业务人员（BD）开通「智能体代理支付」产品，否则调用接口会返回 `ACCESS_FORBIDDEN`（无权限）。**

- 商家自研：应用上线后，联系支付宝 BD 申请开通「智能体代理支付」（委托支付场景另需开通 **App 支付** 产品，供预下单 `alipay.trade.order.prepay` 使用）。
- 服务商代开发：联系 BD 为对应商家开通产品，并在「第三方应用 > 商家授权」添加产品后获取 `app_auth_token`。

无开通权限时的典型报错：预下单 `ACCESS_FORBIDDEN(40006)`、委托支付 `ACQ.ACCESS_FORBIDDEN`。

---

## 4. 上线应用

- 小程序应用：审核通过后自动上线。
- 网页/移动应用：需手动提交审核（约 1 个工作日），上线后才能在生产环境调用。

---

## 5. 集成服务端 SDK

服务端 SDK 封装了签名/验签与 HTTP 请求。官方支持 **Java、PHP、NodeJS、Python、.NET** 五种语言。

### Java（Maven）

```xml
<dependency>
    <groupId>com.alipay.sdk</groupId>
    <artifactId>alipay-sdk-java</artifactId>
    <version>4.40.852.ALL</version>
</dependency>
```

### AlipayClient 初始化（公钥模式，初始化一次后复用）

```java
private AlipayConfig getAlipayConfig() {
    AlipayConfig cfg = new AlipayConfig();
    cfg.setServerUrl("https://openapi.alipay.com/gateway.do"); // 生产网关；沙箱见下
    cfg.setAppId("your_app_id");                 // 第 1 步拿到的 APPID
    cfg.setPrivateKey("your_private_key");       // 第 2 步生成的应用私钥（PKCS8 单行）
    cfg.setAlipayPublicKey("alipay_public_key"); // 第 2 步平台返回的支付宝公钥
    cfg.setFormat("json");
    cfg.setCharset("UTF-8");
    cfg.setSignType("RSA2");
    return cfg;
}
// AlipayClient client = new DefaultAlipayClient(getAlipayConfig());
```

> 服务端 SDK 总览：<https://opendocs.alipay.com/common/02n6z6>
> 可运行的完整示例见本技能包 `examples/sandbox-demo/`。

## 6. 沙箱测试（上生产前先在沙箱验证）

1. 进入 [沙箱控制台](https://openhome.alipay.com/develop/sandbox/app)，获取**沙箱专用 APPID** 与沙箱网关。
2. 沙箱网关：`https://openapi-sandbox.dl.alipaydev.com/gateway.do`。
3. 「沙箱账号」里有买家账号（登录密码/支付密码），用于授权与支付测试。
4. 沙箱验证签名与基础接口连通后，再切换到生产网关与生产密钥。

> 注意：委托支付/AI付为邀测产品，标准沙箱可能未开通对应接口；签名与网关连通可在沙箱验证，接口真正打通需账号进白名单。
