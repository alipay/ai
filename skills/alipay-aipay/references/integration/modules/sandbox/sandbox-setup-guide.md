# 沙箱环境初始化指南

本文档是沙箱环境配置的**单一可信来源**。

---

## 1. 环境说明

### 1.1 快速沙箱优势
- ✅ **零配置启动**：无需登录支付宝开放平台，自动获取测试账号与密钥
- ✅ **即开即用**：全自动配置网关及应用信息
- ✅ **安全隔离**：测试环境不涉及真实资金，可放心调试

### 1.2 Windows 用户注意事项
**快速沙箱 CLI 工具暂不支持 Windows 系统**。

Windows 用户请前往支付宝开放平台（https://open.alipay.com）- 控制台 - 沙箱应用 自行获取沙箱信息。

### 1.3 其他说明
沙箱环境暂不支持用户主动通过看板查看测试账号的交易信息（如：卖家余额、交易状态等）。**严禁**引导用户访问所谓的“支付宝沙箱网页版”查看测试账户的余额变动等。

## 2. 密钥格式说明

| 语言 | 私钥格式 | 使用字段 |
|------|----------|----------|
| Java | PKCS#8 | `appPrivateKey` |
| 非 Java（Python、Node.js、PHP、.NET 等） | PKCS#1 | `appPrivatePkcsKey` |

**⚠️ 防误判提醒**：
- 沙箱配置直接使用沙箱返回的对应私钥字段；生产配置必须确认应用私钥格式与项目语言匹配
- 写入 SDK 配置的私钥值都使用不带 PEM 头尾或其他前后缀的原始密钥字符串，禁止手动添加 PEM 头尾、其他前后缀、包装行或说明文字
- 生产应用私钥格式与当前语言 SDK 要求不一致时，使用支付宝开放平台密钥工具做私钥格式转换后再配置
- 禁止因 `ERR_OSSL_UNSUPPORTED` 推断私钥格式有问题（通常是 SDK 配置参数错误）
- 详见 [SDK 说明文档 - 私钥格式](../alipay-sdk-reminder.md)

## 4. 强校验规则（卡点）

工具返回环境信息后，**必须执行逐项强校验**，校验通过方可进入后续步骤。未通过则**阻断流程**，要求重新创建。

### 4.1 必含字段清单

以下字段必须全部存在且非空：

| 字段 | 说明 |
|---|---|
| `appId` | 沙箱应用 ID |
| `appPrivateKey` / `appPrivatePkcsKey` | 应用私钥（Java 使用 `appPrivateKey`，非 Java 使用 `appPrivatePkcsKey`） |
| `alipayPublicKey` | 支付宝公钥 |
| `appPublicKey` | 应用公钥 |
| 商家账号 | 沙箱商家登录账号 |
| 买家账号 | 沙箱买家测试账号 |

### 4.2 绝对禁止（红线）

**以下行为严格禁止：**

1. **禁止编造字段** —— 不得生成、拼接、模拟、猜测缺失的字段值
2. **禁止用占位符冒充真实数据** —— 当沙箱成功返回完整配置时，禁止用占位符代替真实返回值
3. **禁止假数据** —— 不得自行生成示例私钥、示例账号等任何假数据
4. **禁止假设存在** —— 不得假设"工具返回的应该都有"而不逐项检查
5. **禁止部分通过** —— 不得因"只有 1 个字段缺失"而继续执行
6. **禁止跳过校验** —— 不得在未完成逐项校验的情况下直接使用返回结果

### 4.3 校验执行清单

**规则：** 任一字段缺失 → **立即停止** → 输出错误提示 → **禁止继续**

```
□ 检查 appId
  IF 缺失：输出 "❌ 沙箱环境校验失败：appId 为空，无法继续集成，请重新创建沙箱环境。" → STOP

□ 检查应用私钥（根据语言选择）
  - Java：检查 `appPrivateKey`（PKCS#8）
  - 非 Java：检查 `appPrivatePkcsKey`（PKCS#1）
  IF 缺失：输出 "❌ 沙箱环境校验失败：应用私钥字段为空，无法继续集成，请重新创建沙箱环境。" → STOP

□ 检查 alipayPublicKey / appPublicKey / 商家账号 / 买家账号
  IF 任一缺失：输出对应字段缺失错误 → STOP

□ 全部通过 → 输出 "✅ 沙箱环境校验通过"
```

### 4.3.1 配置写入后的后置校验

后续步骤将沙箱配置写入用户项目后，必须立即做配置准确性后置校验；校验失败时停止联调，先重新复制沙箱字段或修正 SDK 配置。必须确认以下目标：

- 确认写入项目的 `appId`、应用私钥、支付宝公钥来自同一份沙箱返回配置，且与沙箱返回配置一致。
- 确认应用私钥不是占位符、示例值或其他来源的密钥，未出现复制截断或粘贴污染。
- 检查 SDK 配置与私钥字段匹配：`appPrivatePkcsKey` 对应 PKCS#1，`appPrivateKey` 对应 PKCS#8。
- 检查写入 SDK 配置的私钥值不包含 PEM 头尾、其他前后缀、包装行或说明文字。
- 校验输出不得包含私钥、公钥、签名串或完整支付表单。

**常见失败处理：**
- 私钥不完整、配置不一致或 SDK 初始化/解析失败：重新复制完整私钥字段，禁止补字符、猜测或格式转换。
- `DECODER routines::unsupported`：优先检查私钥是否复制完整、是否错误添加 PEM 头尾或其他前后缀、`keyType` 是否错误，不要切换成 PKCS#8 试错。

### 4.4 重试与失败处理

| 失败场景 | 处理策略 |
|---------|---------|
| 首次失败 | 提示 "❌ 快速沙箱创建失败，请稍后重试"，等待用户指示后重试 |
| 字段缺失 | 明确告知**具体缺失字段名称**，禁止只说"部分字段缺失" |
| 反复失败（≥2次） | 输出强提醒，提供两个选项：①重新输入指令 ②自行申领沙箱应用，禁止继续自动重试 |
| 用户提供手动配置 | 仍需按「校验执行清单」逐项校验，通过后方可继续 |

**反复失败提示模板**：
```

缺失字段：[列出具体缺失的字段名称]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
请您选择以下方式之一解决：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【选项一】重新输入创建快速沙箱指令
  操作方式：在对话中重新输入"创建快速沙箱"或类似指令
  适用场景：可能因服务波动导致创建失败，稍后重试可能成功

【选项二】自行申领沙箱应用
  操作方式：前往支付宝开放平台（https://open.alipay.com）手动申领沙箱应用
```
