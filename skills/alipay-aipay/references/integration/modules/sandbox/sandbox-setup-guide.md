# 沙箱环境初始化指南

本文档是沙箱环境配置的**单一可信来源**。

---

## 1. 环境说明

### 1.1 快速沙箱优势
- ✅ **零配置启动**：无需登录支付宝开放平台，自动获取测试账号与密钥
- ✅ **即开即用**：全自动配置网关及应用信息
- ✅ **安全隔离**：测试环境不涉及真实资金，可放心调试

### 1.2 Windows 用户注意事项
**快速沙箱 CLI 工具暂不支持 Windows 系统**。

Windows 用户请前往支付宝开放平台（https://open.alipay.com）- 控制台 - 沙箱应用，自行获取沙箱信息。

### 1.3 其他说明
沙箱环境暂不支持用户主动通过看板查看测试账号的交易信息（如：卖家余额、交易状态等）。**严禁**引导用户访问所谓的“支付宝沙箱网页版”查看测试账户的余额变动等。

## 2. 密钥格式说明

| 语言 | 私钥格式 | 使用字段 |
|------|----------|----------|
| Java | PKCS#8 | `appPrivateKey` |
| 非 Java（Python、Node.js、PHP、.NET 等） | PKCS#1 | `appPrivatePkcsKey` |

**⚠️ 防误判提醒**：
- 沙箱配置直接使用沙箱返回的对应私钥字段；生产配置必须确认应用私钥格式与项目语言匹配
- 写入 SDK 配置的私钥值都使用不带 PEM 头尾或其他前后缀的原始密钥字符串，禁止手动添加 PEM 头尾、其他前后缀、包装行或说明文字
- 生产应用私钥格式与当前语言 SDK 要求不一致时，使用支付宝开放平台密钥工具做私钥格式转换后再配置
- 禁止因 `ERR_OSSL_UNSUPPORTED` 推断私钥格式或运行时不兼容；应先检查复制污染、字段映射和 SDK 配置参数
- 详见 [SDK 说明文档 - 私钥格式](../alipay-sdk-reminder.md)

## 3. 沙箱初始化流程

### 3.1 获取候选沙箱配置

- Unix/macOS/Linux：参考 [alipay-sandbox-tool](alipay-sandbox-tool.md) 调用快速沙箱工具。
- Windows：快速沙箱 CLI 工具暂不支持，按第 1.2 节引导用户自行获取沙箱信息。
- 工具返回 `success === true` 只代表拿到候选 `data`；必须完成第 4 节字段完整性校验后，才允许写入正式配置文件。
- `success !== true`、`data` 缺失或字段不完整时，按第 6 节失败处理阻断流程。

### 3.2 固定处理顺序

1. 提取候选 `data`。
2. 按第 4 节逐项校验必含字段。
3. 校验通过后，将候选 `data` 按原始 JSON 结构写入正式本地配置文件。
4. 输出第 5 节规定的关键字段摘要和实际配置文件路径。
5. 后续写入项目 `.env` 或 SDK 配置时，统一从该本地 JSON 文件读取并映射，避免在对话中复述密钥等长字段。

校验和落盘可使用 Agent 的结构化 JSON 处理能力、Node.js/Python 标准库或其他可用工具实现，不强依赖 `jq`。

## 4. 字段完整性校验（卡点）

工具返回环境信息后，必须先逐项校验。任一必含字段缺失或为空时，立即停止，不得写入正式 `.alipay-sandbox.json`，并明确告知具体缺失字段。

### 4.1 必含字段清单

以下字段必须按项目语言和测试账号要求存在且非空：

| 字段 | 说明 |
|---|---|
| `appId` | 沙箱应用 ID |
| 应用私钥 | Java 必须存在 `appPrivateKey`；非 Java 必须存在 `appPrivatePkcsKey` |
| `alipayPublicKey` | 支付宝公钥 |
| `appPublicKey` | 应用公钥 |
| 商家账号 | 沙箱商家登录账号 |
| 买家账号 | 沙箱买家测试账号 |
| 买家账号 `logonPassword` | 沙箱买家登录密码 |
| 买家账号 `payPassword` | 沙箱买家支付密码 |

### 4.2 校验执行清单

**规则：** 任一字段缺失 → **立即停止** → 输出错误提示 → **禁止继续**

```
□ 检查 appId
  IF 缺失：输出 "❌ 沙箱环境校验失败：appId 为空，无法继续集成，请重新创建沙箱环境。" → STOP

□ 检查应用私钥（根据语言选择）
  - Java：检查 `appPrivateKey`（PKCS#8）
  - 非 Java：检查 `appPrivatePkcsKey`（PKCS#1）
  IF 缺失：输出 "❌ 沙箱环境校验失败：应用私钥字段为空，无法继续集成，请重新创建沙箱环境。" → STOP

□ 检查 alipayPublicKey / appPublicKey / 商家账号 / 买家账号 / 买家账号 logonPassword / 买家账号 payPassword
  IF 任一缺失：输出对应字段缺失错误 → STOP

□ 全部通过 → 不对客输出字段核对表、字段列表、独立的“字段完整性校验通过”提示或任何“沙箱测试通过”表述；直接进入落盘，并按第 5.2 节只输出一张沙箱环境摘要表。
```

## 5. 落盘与摘要输出

### 5.1 本地配置文件

- 默认写入已确认的目标项目根目录 `.alipay-sandbox.json`；若该目标目录不适合写入，则写入系统临时目录 `alipay-sandbox-<timestamp>.json`，不得改写到 Agent 原始工作目录或其他项目。
- JSON 文件必须保留沙箱返回的原始字段名和嵌套结构，不要转换成 `.env` 扁平键值。
- 文件权限应尽量限制为当前用户可读写，例如 `chmod 600`。
- 配置落盘后，必须在对话中输出实际路径。路径必须可定位，例如 `/path/to/project/.alipay-sandbox.json` 或 `./.alipay-sandbox.json`。

### 5.2 对话摘要

字段无缺失并完成落盘后，对客只用下方一张表展示沙箱信息，不再展示字段核对表、已校验字段列表或独立的校验结论。表中包含本地配置文件路径、关键标识，以及工具实际返回的沙箱测试账号登录密码和支付密码；密钥、完整签名串和完整支付表单等长字段不得展开。账号没有返回某一可选字段时省略对应内容，禁止编造。

```
沙箱环境摘要

| 信息 | 内容 |
|---|---|
| 配置文件 | `<本地配置文件路径>` |
| 应用 | `appId=<appId>[, pid=<pid，如有>]` |
| 商家账号 | `email=<email，如有>, userId=<userId，如有>[, 登录密码=<logonPassword，如有>][, 支付密码=<payPassword，如有>]` |
| 买家账号 | `email=<email，如有>, userId=<userId，如有>, 登录密码=<logonPassword>, 支付密码=<payPassword>` |
| 沙箱 | `sandboxId=<sandboxId，如有>, sandboxName=<sandboxName，如有>` |
| 密钥字段 | `appPrivateKey/appPrivatePkcsKey/alipayPublicKey/appPublicKey` 已写入本地配置文件，默认不在对话中展开 |
```

这张表表示沙箱配置字段完整且已经落盘，不表示支付链路已经测试。禁止把表名、前后提示或后续总结改写为“沙箱环境校验通过”“沙箱测试通过”“沙箱环境测试通过”等结论。

### 5.3 成功创建后提示（卡点）

输出第 5.2 节摘要后，必须继续输出以下成功提示。提示可以和摘要连续输出，但不得省略关键提醒。

```
⚠️ 快速沙箱环境已创建完成

📌 环境说明
- 当前快速沙箱环境仅供测试，不涉及真实资金
- 完整沙箱信息已写入本地配置文件；对话中展示工具实际返回的沙箱测试账号登录密码和支付密码，以便完成沙箱付款，不展开私钥、公钥、完整签名串或完整支付表单等长字段
- 切勿在项目日志、对话总结或测试结果中打印私钥信息
- 切勿提交正式环境、沙箱环境信息至公共代码仓库（GitHub、GitLab 等）

🔔 必做操作
- 请自行妥善保管本地配置文件、密钥及测试账号等信息
- 若需要迁移配置，请从本地配置文件复制，避免通过对话记录传递密钥等完整长字段

🔒 安全提醒
- 正式上线前请确保您已拥有正式应用的 appId、对应的应用私钥及支付宝公钥，并确认应用和支付产品已达到上线所需状态后切换至正式环境
- 替换生产配置或启动生产支付测试前，确认 `appId`、应用公钥、应用私钥属于同一套生产应用密钥，并根据项目语言确认私钥格式正确；格式不匹配时使用支付宝开放平台密钥工具转换
- 正式环境网关地址：https://openapi.alipay.com/gateway.do
- 正式环境涉及真实资金交易，正式上线前应在沙箱环境完成所有可验证的支付链路；受本地网络条件限制的项目必须另行标记和验证
```

### 5.4 网站支付沙箱付款提示（代码生成后、集成收尾前）

网站支付代码生成和配置后置校验完成后，先从已确认的目标项目脚本、路由和运行配置中确定真实的服务启动命令和支付页面访问地址，再按回跳分支处理：

- **默认分支**：用户未明确关闭同步回跳时，确认 SDK 实际使用的 `return_url`。启动服务并 GET 访问不带支付结果参数的该地址，只验证路由和页面壳；无订单上下文时应展示安全的中性状态，不得放宽真实回跳请求的验签和订单归属校验。确认无 404、认证或重定向循环，且单页应用直接刷新可用。
- **渲染验证**：Agent 具备浏览器/UI 能力时，打开回跳页确认非空白、无可见报错且关键内容正常渲染；不具备时，请用户打开精确地址确认。未取得证据时不得宣称回跳页已验证或网站支付集成完成。
- **关闭分支**：用户已明确关闭同步回跳时，不要求或猜测 `return_url`；确认 SDK 请求未传该字段，并验证异步通知、交易查询和商户订单查询页。

完成对应分支校验后，使用本次已校验沙箱配置中的真实买家账号输出以下提示。不得使用示例端口、占位 URL 或猜测地址；启动命令、支付页面访问地址或账号字段无法确定时，指出缺失项并标记为人工待补充。下方尖括号仅是模板槽位，对客输出时必须替换为真实值；缺少真实值时删除对应占位内容并说明缺失项。

`curl` 调用下单接口并取得 HTML 支付表单只属于接口冒烟检查，不能替代用户付款指引，也不能据此标记步骤 5 完成。必须提供用户可在浏览器中打开的支付页面地址；页面打开沙箱收银台后，以下两种体验方式同时告知用户，由用户自行二选一。

```text
🧪 网站支付沙箱付款指引

网站支付已接入沙箱环境，可以发起一笔测试付款：
1. 启动服务并保持运行：`<从目标项目确定的真实启动命令>`
2. 使用浏览器访问支付页面：`<从目标项目确定的真实访问地址>`
3. 在页面中发起支付，确认页面成功唤起支付宝沙箱收银台。

付款方式（二选一）：

方式一：收银台账号登录
4. 在沙箱收银台使用以下沙箱买家账号和登录密码登录，无需使用真实支付宝账号：
   - 买家账号：<email>
   - 登录密码：<logonPassword>
5. 登录后输入沙箱支付密码完成核身与付款：
   - 支付密码：<payPassword>

方式二：安卓沙箱支付宝扫码
4. 沙箱支付宝仅支持 Android。复制以下官方地址到网页浏览器打开，按页面指引安装沙箱支付宝：
   https://mdn.alipayobjects.com/sandboxsys/afts/img/_itHRrdOD9oAAAAAAAAAAAAADgSLAQBr/original
5. 在沙箱支付宝中使用上述沙箱买家账号和登录密码登录。
6. 返回浏览器中的沙箱收银台，使用沙箱支付宝扫描页面二维码，再输入上述沙箱支付密码完成付款。
```

紧接上述步骤追加结果判断，必须按实际回跳分支二选一，禁止同时输出两项：

- **默认分支**：“付款后确认浏览器自动返回商户结果页，页面可正常展示并通过服务端查询呈现订单状态。”
- **关闭分支**：“付款后不会自动返回商户网站；请手动返回商户订单查询页并刷新或查询订单状态。”

两个分支都必须继续提醒：前台页面或同步回跳参数不是支付成功依据，最终结果以验签通过的异步通知或交易查询结果为准。异步通知只有在 `notify_url` 可被支付宝服务器公网访问时才能联调；当前仅有本地或局域网地址时，将该项标记为人工待验证。

**下载渠道红线：**
- 严禁声称 iOS 支持安装沙箱版支付宝。
- 严禁引导用户前往应用市场、搜索引擎或第三方站点搜索下载。
- 如果上述直链或页面二维码无法使用，只能引导用户前往支付宝开放平台沙箱工具页 `https://open.alipay.com/develop/sandbox/tool`，按页面指引安装。

## 6. 红线与失败处理

### 6.1 绝对禁止

1. 禁止编造、拼接、模拟、猜测缺失字段。
2. 禁止用占位符或示例假数据冒充真实返回值。
3. 禁止假设“工具返回的应该都有”而跳过逐项校验。
4. 禁止字段不完整时继续集成或写入正式 `.alipay-sandbox.json`。
5. 除沙箱测试账号的登录密码和支付密码外，禁止把密钥等长字段整段刷屏输出；禁止展示真实生产账号密码。

### 6.2 重复创建红线

- 候选 `data` 通过第 4 节字段完整性校验后，必须使用本次返回的 `data` 完成正式落盘。
- 禁止为了“把 data 写入本地文件”“避免对话展示长字段”或“重新拿一份完整配置”再次创建沙箱。
- 如果第一次通过校验的 `data` 因 Agent 操作失误未落盘，应先从已有工具输出、过程产物或当前可访问的执行结果中恢复；无法恢复时必须向用户说明问题并征得用户明确同意后，才可重新创建。

### 6.3 重试与失败处理

| 失败场景 | 处理策略 |
|---------|---------|
| 首次创建失败 | 提示 "❌ 快速沙箱创建失败，请稍后重试"，等待用户指示后重试 |
| 字段缺失 | 明确告知具体缺失字段名称，禁止只说"部分字段缺失" |
| 反复失败（≥2次） | 输出强提醒，提供两个选项：①重新输入指令 ②自行申领沙箱应用，禁止继续自动重试 |
| 用户提供手动配置 | 仍需按第 4 节逐项校验，通过后方可继续 |

反复失败时输出：

```
🚨 快速沙箱环境多次创建失败/字段不完整

缺失字段：[列出具体缺失的字段名称]

请您选择以下方式之一解决：
1. 重新输入“创建快速沙箱”指令
2. 前往支付宝开放平台（https://open.alipay.com）自行申领沙箱应用，需要使用自己的沙箱账号，获取 appId、应用私钥、支付宝公钥等信息后手动配置

在配置完整之前，无法继续集成流程。
```

## 7. 写入项目后的后置校验

后续步骤将沙箱配置写入用户项目后，必须立即做配置准确性后置校验；校验失败时停止联调，先从第 5.1 节的可信配置文件重新读取字段或修正 SDK 配置。

- 确认写入项目的 `appId`、应用私钥、支付宝公钥来自同一份沙箱返回配置。
- 从第 5.1 节输出的实际配置文件结构化读取并映射字段，禁止从对话、历史代码或示例值复制密钥。
- 以不输出字段原文的方式比较项目实际读取值与沙箱原值，确认应用私钥不是占位符、示例值或其他来源的密钥，且未出现截断、字符替换或粘贴污染。
- 检查 SDK 配置与私钥字段匹配：`appPrivatePkcsKey` 对应 PKCS#1，`appPrivateKey` 对应 PKCS#8。
- 检查写入 SDK 配置的私钥值不包含 PEM 头尾、其他前后缀、包装行或说明文字；业务协议直接使用原生密码库签名时，可按密码库要求临时适配调用入参，但不得修改或覆盖原始配置值。
- 仅按量付费沙箱联调：项目沙箱运行配置中的 `serviceId` 固定为 `api_mock_service_id`，不从快速沙箱返回中寻找，也不向用户索要正式 `serviceId`。该值不得进入生产配置。
- 校验输出不得包含私钥、公钥、签名串或完整支付表单。

**常见失败处理：**
- 私钥不完整、配置不一致或 SDK 初始化/签名失败：从第 5.1 节的可信配置文件重新读取正确字段，禁止补字符、猜测或格式转换。
- `ERR_OSSL_UNSUPPORTED`、`DECODER routines::unsupported` 等错误：优先检查项目读取值是否与沙箱原值一致、字段映射是否正确、SDK 配置是否被加工；不要切换私钥字段或在缺少对照证据时归因于运行时、OpenSSL 或 SDK 兼容性。
