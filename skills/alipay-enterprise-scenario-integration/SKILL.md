# 支付宝企业码场景接入标准方案 Skill

## 定位

本 Skill 是企业码单场景接入的方案编排器。每次任务只接入一个费用场景，底层能力由三个同级子 Skill 提供：

- `alipay-enterprise-ec`：企业入驻、员工签约、企业和员工管理
- `alipay-enterprise-expense-control`：制度、额度及外部费控 SPI
- `alipay-enterprise-bill`：账单、订单和对账

本 Skill 只负责场景决策、模块编排、跨域聚合与最终校验，不重复维护子域接口字段。

## 文档分工

- 本文件：任务入口、默认模块、阶段顺序和最终校验入口
- [场景接入指南](references/scenario-integration-guide.md)：集成准备、场景决策、方案范围和交付要求
- [场景决策规则](references/scenario-decision-rules.md)：费用类型、子类、因公场景、规则因子和因公优先决策
- [多 Agent 代码生成编排规则](references/multi-agent-codegen.md)：子 Agent 加载、写入边界和完成屏障
- [主方案聚合质量门禁](references/quality-gates/aggregate.md)：场景文件、跨域工程和消息入口校验
- [已有项目衔接契约](references/integration-contract.md)：增量接入时的三域衔接证据

## 任务形态

- 方案设计：输出场景决策、模式、模块、接口清单、规则配置和验证要点，不生成代码。
- 代码生成：场景和方案范围明确后生成或修改代码，并执行 SDK 预检、分域生成和工程校验。

## 场景闸门

1. 每次只允许一个场景。用户一次提出多个场景时，必须让用户选择本次先接入哪一个。
2. 用户明确费用类型、费用子类和因公场景时直接沿用，并按文档校验合法性。
3. 上下文可唯一推断时，展示推断结果后继续，不重复询问。
4. 存在多个合法选择，或必用规则因子缺少业务值时，必须询问用户。示例：地铁必须确认城市或具体 `CARD_TYPE`。
5. 文档找不到的场景、枚举、规则因子或值不得自行补成 `DEFAULT`。
6. 代码生成前必须生成 `<项目>/.alipay-skill/scenario.json`，且不得保留 `NEEDS_USER_CONFIRM`。

## 默认方案

默认采用企业码标准方案的三域基础模块：

| 能力域 | 必选模块 | 按需模块 |
|---|---|---|
| 员企 | 企业入驻、员工签约、员工管理、企业管理 | 部门、核算主体、企业地址、企业消息任务 |
| 费控 | 制度管理 | 手工发放、额度管理 |
| 账单 | 账单管理 | 订单同步、对账单下载 |

- 员企默认邀请企业注册、邀请员工签约。
- 费控不默认内部或外部；无法从上下文推断时必须询问。
- 账单默认推模式。
- Java 默认 WebSocket；非 Java 默认 HTTP(S)，除非用户环境和文档明确支持其他方式。
- 场景依赖订单履约明细时，订单同步应纳入范围；是否必选由场景接入指南和账单文档决定。

## 子 Skill 与多 Agent

代码生成必须先读取 [多 Agent 代码生成编排规则](references/multi-agent-codegen.md)，优先启动员企、费控、账单分域子 Agent，并显式加载对应子 Skill。未完成启动或降级确认前，不得读取接口文档或生成接口调用代码。

如果同级子 Skill 不存在，可从本 Skill 的 `subskills/*.zip` 解压到 skills 根目录。

## 执行阶段

1. 场景决策：读取场景接入指南、场景决策规则以及必要的费控/账单枚举文档，确认单一场景。
2. 项目判断：确认新工程或已有项目；已有项目先输出盘点、增量计划和拟定衔接契约，等待用户确认。
3. SDK 预检：生成接口调用代码前完成对应语言 SDK 或 HTTP(S) 接入方式预检。
4. 员企阶段：员企 Agent 按子 Skill 生成并运行本域 validator。
5. 费控阶段：费控 Agent 沿用 `scenario.json` 的场景决策生成制度和所选模块，并运行本域 validator。
6. 账单阶段：账单 Agent 按场景识别字段和所选模块生成代码，并运行本域 validator。
7. 消息和聚合阶段：所有子 Agent 返回最终回执后，主 Agent 统一处理公共配置、消息入口和主聚合校验。

用户只选择基础模块时，不得读取或实现未选择的扩展模块。

## 生成后校验

```bash
node alipay-enterprise-scenario-integration/scripts/validate_codegen.js <生成项目目录>
```

主脚本调用三个子域 validator，并读取 `.alipay-skill/scenario.json` 校验所选费用类型、子类、`scene_type`、必用规则因子、因公优先配置和账单场景识别。

退出码 `0` 表示通过，`1` 表示生成代码不符合门禁，`2` 表示主门禁或子 validator 执行不可信。出现 `2` 时不得宣布生成完成。
